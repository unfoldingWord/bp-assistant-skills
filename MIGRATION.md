# Migration to AWS Chainguard

Complete checklist for standing up the bot on a new host.

## What you need

| Item | Source | How to migrate |
|------|--------|----------------|
| App git repo | `/srv/bot/app/` | `git clone` |
| Workspace git repo | `/srv/bot/workspace/` | `git clone` |
| Hand-curated data | workspace/data + app/data + output | tar bundle (step 1) |
| Config + secrets | `/srv/bot/config/` | scp (step 2) |
| App local config | `app/config.local.json` | scp (step 2) |
| Claude auth | `claude-config` Docker volume | Re-authenticate on new host (step 5) |
| Docker networks | `work-net`, `home-net` | Create on new host (step 4) |
| Published data | Door43 + Google | Curation tool fetches it (step 6) |

## 1. Create and transfer data bundle (on current server)

```bash
cd /srv/bot

# Bundle hand-curated data (~30M compressed)
tar czf /tmp/workspace-migrate.tar.gz \
  -C /srv/bot/workspace \
  data/editor-feedback data/editor-notes data/editor-ult data/editor-ust data/editor-ult-english \
  data/quick-ref data/en_tw data/ta-flat \
  data/abstract_nouns.txt data/books.txt data/translation-issues.csv data/tw_headwords.json \
  output/ \
  -C /srv/bot \
  app/data/

scp /tmp/workspace-migrate.tar.gz user@new-aws-host:/tmp/
```

### What's in the bundle

| Dir/File | Why | Size |
|----------|-----|------|
| `data/editor-*` | Human editor feedback and corrections | ~12M |
| `data/quick-ref/` | Accumulated ULT/UST rendering decisions (CSV) | ~52K |
| `data/en_tw/` | Consolidated Translation Words (manually curated from Door43) | ~4M |
| `data/ta-flat/` | Flattened Translation Academy | ~2M |
| `data/abstract_nouns.txt` | Hand-curated word list for issue detection | static |
| `data/books.txt` | OT book code list | static |
| `data/translation-issues.csv` | All 94 issue types (master reference) | static |
| `data/tw_headwords.json` | Translation Words headword lookup | ~68K |
| `output/` | Generated ULT, UST, notes, issues, reviews (built upon) | ~14M |
| `app/data/` | Sessions, metrics, pending merges, checkpoints | ~184K |

### What's NOT in the bundle (regenerated by curation tool)

- `data/published_ult/`, `data/published_ust/`, `data/published-tns/` -- fetched from Door43
- `data/published_ult_english/`, `data/published_ust_english/` -- extracted from aligned USFM via usfm-js
- `data/hebrew_bible/`, `data/t4t/` -- fetched from Door43
- `data/glossary/` -- fetched from Google Sheets
- `data/templates.csv`, `data/issues_resolved.txt` -- fetched from Google Sheets/Docs
- `data/cache/` -- rebuilt by index builders
- `door43-repos/` -- shallow clones created at runtime by door43-push

```bash
# Config directory (secrets, .env, model provider config)
scp -r /srv/bot/config/ user@new-aws-host:/srv/bot/config/

# Gitignored app config (admin/authorized user IDs)
```

## 3. Clone repos and unpack data (on new host)

```bash
mkdir -p /srv/bot
cd /srv/bot

# Clone repos
git clone <app-repo-url> app
git clone <workspace-repo-url> workspace

# Unpack curated data
tar xzf /tmp/workspace-migrate.tar.gz

# Verify
ls workspace/data/editor-ult/
ls workspace/output/AI-ULT/
ls app/data/sessions/
```

## 4. Create Docker infrastructure

```bash
# Create external networks
docker network create work-net
docker network create home-net

# Create the Claude config volume (will be populated in step 5)
docker volume create bot_claude-config
```

## 5. Authenticate Claude

The `claude-config` Docker volume stores Claude OAuth tokens. On the new host, either:

**Option A: Re-authenticate fresh**
```bash
# Build and start the container first
cd /srv/bot/app
docker compose build
docker compose up -d

# Then trigger auth (the bot will prompt on first Claude SDK call,
# or you can manually authenticate via the volume)
```

**Option B: Copy volume data from old host**
```bash
# On old host
docker run --rm -v bot_claude-config:/data -v /tmp:/backup alpine tar czf /backup/claude-config.tar.gz -C /data .

scp /tmp/claude-config.tar.gz user@new-aws-host:/tmp/

# On new host
docker run --rm -v bot_claude-config:/data -v /tmp:/backup alpine sh -c "cd /data && tar xzf /backup/claude-config.tar.gz"
```

## 6. Build, start, and populate published data

```bash
cd /srv/bot/app
docker compose build
docker compose up -d

# Check logs
docker logs zulip-bot --tail 30 -f
```

Once the bot is running, DM it in Zulip to populate all fetchable data:

```
setup data
```

This runs the curation tool (`step: "setup"`, force mode) which:
1. Discovers published OT books from Door43 releases API (currently 25 books, v88)
2. Fetches aligned ULT, UST, TN, Hebrew Bible, T4T for each book
3. Fetches glossary, templates, and issues_resolved from Google Sheets/Docs
4. Extracts unaligned English ULT & UST (via usfm-js proper USFM parser)
5. Resolves any missing GL quotes on TNs
6. Builds Strong's index, UST index, and TN index in `data/cache/`

The bot will DM you a summary when complete.

## 7. Verify

DM the bot:
```
update data check
```

This reports the current release tag, book count, and whether any new books are available.

## 8. Ongoing updates

**Admin DM commands:**
```
update data              # Normal update (cache-aware, only fetches if stale)
update data force        # Force refetch everything
update data check        # Dry run — report what's changed
update data fetch-door43 # Just Door43 fetch step
update data build-indexes # Just rebuild indexes
setup data               # Alias for full force (same as initial setup)
```

**MCP tool** (available to Claude in pipelines):
```
curate_published_data({ step: "check" })
curate_published_data({ force: true })
curate_published_data({ step: "build-indexes" })
```

Run monthly or after a new Door43 release (releases are infrequent).

## Notes

- Everything runs inside the Docker container -- no host-side Node.js, Python, or bash needed
- `usfm-js` is an app dependency, baked into the Docker image at build time
- `/workspace` is bind-mounted, so fetched data persists across container rebuilds
- The standalone `workspace/.claude/skills/utilities/scripts/curate-published-data.mjs` is also available for host-side use if you have Node.js installed
